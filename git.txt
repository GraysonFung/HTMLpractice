--------版本库操作
$ git init ==>master主分支   初始化版本库
$ git status		查看版本库状态
$ git add readme.txt	添加文件到 暂存区
$ git commit -m "create readme.txt"  提交文件到分支,-m  后面是本次提交的说明
$ git diff readme.txt	查看改变内容

$ git log 			查看日志（详细版），以便确定回到哪个版本
$ git log --pretty=online	查看日志

$ git reset --hard HEAD^	回退到上个版本
$ git reset --hard HEAD^^ 	回退到上上个版本
$ git reset --hard HEAD~100 	回退到前100个版本
$ git reset --hard commit_id	回退到某版本
$ git checkout -- readme.txt	迁出，从暂存区或者文件（暂存区为空）中迁出，迁出以后都会影响工作区的原始文件

$ git reflog  记录你的每一次命令

**撤销修改
$ git checkout -- file  把file文件在工作区的修改全部撤销,回到之前状态。总之，就是让这个文件回到最近一次git commit或git add后的状态。
$ git reset HEAD file 可以把暂存区的修改撤销掉（unstage），重新放回工作区

**删除文件
$ rm file     删除工作区的file文档
	$ git rm file 并且$ git commit;删除版本库里的file文件    
	$ git checkout -- file   删除恢复（误删）



--------远程仓库
**添加远程库
$ git remote add origin git@github.com:GraysonFung/learngit.git
$ git push -u origin master    第一次提交，把本地master分支的最新修改推送至GitHub
$ git push origin master	后期提交
	提示错误：fatal: remote origin already exists.

			1、先删除远程 Git 仓库

 			   $ git remote rm origin

			2、再添加远程 Git 仓库

------分支管理
**创建与合并分支
$ git checkout -b dev	git checkout 命令加上 -b参 数表示创建并切换，相当于以下两条命令：
$ git branch dev	创建分支dev
$ git checkout dev	切换到dev分支

$ git branch    查看目前所有的分支，当前分支前面会标一个*号

$ git merge dev		git merge命令用于合并指定分支（dev）到当前分支(*)。

$ git branch -d dev	删除dev分支,合并完成后，就可以放心地删除dev分支了




$ git branch   查看目前所有的分支   开发分支dev 调试分支bug  合并分支
$ git git branch dev 	创建分支
$ git checkout dev	查看分支
$ git checkout master	切回分支
$ git merge dev		合并分支   在master合并dev
$ git btranch -D dev	删除分支，D强制删除，有没有提交都可以删除；d提交后才能删除
